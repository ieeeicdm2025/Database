实验指南(七)：Flink

Flink（官网）

1、简介

Apache Flink 是为分布式、高性能、随时可用以及准确的流处理应用程序打造的开源流处理框架，用于对无界和有界数据流进行有状态计算。



2、Flink的核心概念

（1）批处理（DataStream API）与流处理（DataSet API）

  • 流（Stream）：Flink 最主要的处理对象是数据流，即连续不断到达的数据。流数据可以是无界的，意味着数据没有固定的终止点，流会持续不断地生成数据。Flink会不断地处理这些数据，直到程序停止。
  • 批（Batch）：Flink也支持批处理。批处理是对有限数据集的处理，可以视作有限的流。Flink 的批处理是基于流处理的优化实现的。

（2）Flink的结构

  • Client：客户端，提交作业
  • JobManager：负责作业的调度、状态管理和故障恢复。每个作业只有一个 JobManager
  • TaskManager：负责具体任务的执行。Flink 的任务是并行的，每个 TaskManager 运行多个任务槽（task slot）

（3）操作符

Flink 的流处理基于一系列的操作符，这些操作符作用于数据流中的元素。常见的操作符包括：

  • Map：对数据流中的每个元素进行转换。
  • Filter：过滤掉不符合条件的元素。
  • Filter：过滤掉不符合条件的元素。
  • KeyBy：基于某个字段进行分组，类似于 groupBy。
  • Reduce：对数据流中的元素进行归约操作，通常用于聚合计算。
  • Window：将数据流分割成小的时间窗口进行聚合或处理。

（4）窗口

窗口是流处理中用于分组和聚合事件的机制。Flink 支持多种窗口类型：

  • 滚动窗口（Tumbling Window）：不重叠的时间段窗口，数据每到达一个新的时间段就会重新计算。
  • 滑动窗口（Sliding Window）：重叠的时间段窗口，数据在滑动窗口中会不断变化。
  • 会话窗口（Session Window）：基于事件的空闲时间来划分窗口，如果事件之间的时间间隔超过设定值，窗口会关闭。

3、使用minikube部署Flink

（1）使用minikube部署Flink时，需要先使flink可以引用自身

minikube ssh 'sudo ip link set docker0 promisc on'

（2）向minikube中导入所需的镜像

minikube image load flink:scala\_2.12-java8
  • 进入minikube查看是否导入
minikube ssh
docker images


（3）使用flink-cluster.yaml文件部署Flink

kubectl apply -f flink-cluster.yaml
  • 查看pod状态，确认已正常运行
kubectl get pods
  • 程序解释：dataloader-deployment-5d88f4dc7f-85qp9这个pod用于随机生成点数据并监听9000端口，而下面的两个pod即为Flink的jobmanager和taskmanger，在其上运行的程序初始化时先随机生成100个矩形，并为它们建立一棵rtree以优化查询，然后接收dataloader-deployment-5d88f4dc7f-85qp9所生成的点数据，将这收到的点数据与所有举行做contain查询，最终返回矩形的id以及它包含的点个数
  • 可以使用下面的命令查看dataloader-deployment是否在监听9000端口
kubectl logs deployment/dataloader-deployment
  • 使用kubectl的端口转发功能，以访问Flink的webui，这个转发命令行界面需要一直保持这个转发状态，停止转发后就无法访问webui了
kubectl port-forward deployment/flink-jobmanager 8081:8081
  • 在浏览器输入下面的url以访问Flink的webui

http://localhost:8081/



（4）运行程序

  • 将程序所需的jar包复制到jobmanager的pod内，命令中的flink-jobmanager-7f4b868b9-vcjbb需要用自己的jobmanager所在的pod代替
kubectl cp data-streamer-jar-with-dependencies.jar flink-jobmanager-7f4b868b9-vcjbb:/opt/flink
  • 进入jobmanager所在pod的命令行bash
kubectl exec -it flink-jobmanager-7f4b868b9-vcjbb -- /bin/bash
  • 使用下面的语句运行程序
./bin/flink run -m localhost:8081 -c main.DataStreamer /opt/flink/data-streamer-jar-with-dependencies.jar
  • 因为默认情况下jobmanager/taskmanager进程的STDOUT已经输出到console，直接发送到容器的日志系统，是没有办法通过webui查看STDOUT输出的，但是可以通过kubectl logs来查看（这里的flink-taskmanager-5c65765575-zcq7t同样需要替换为自己的taskmanager所在的pod）：
kubectl logs flink-taskmanager-5c65765575-zcq7t
