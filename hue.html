实验指南(十)：Hue

Hue（官网）

1、简介

Hue 提供一个基于浏览器的用户界面，用户可以通过它与 Hadoop 系统进行交互，执行查询、管理数据和配置系统等任务。它使得大数据的使用者，尤其是非技术用户，可以通过图形化的方式完成各种操作，降低了学习和使用的门槛。

2、hue 集成多种 Hadoop 生态系统工具

  • HDFS：文件管理界面，支持文件浏览、上传、下载、删除等操作。
  • Hive：SQL 查询界面，允许用户通过 SQL 查询语言对数据进行操作和分析。
  • Impala：与 Hive 类似，提供 SQL 查询功能，但它是一个更加高效的实时查询引擎。
  • HBase：图形化的 HBase 界面，支持表的查看、编辑等操作。
  • HBase>：一个支持全文搜索的服务，Hue 提供对 Solr 搜索引擎的图形化管理界面。

（3）测试 hue 的功能

  • 登录以后，我们可以在左侧看到当前hive的数据库只有一个默认的数据库，我们在中间的方框处对 hive 进行查询
  • 首先我们创建数据库example_db
CREATE DATABASE IF NOT EXISTS example_db;
  • 这里我们可能会遇到下面红色框里的警告，可以忽视这个警告，等待一会可以看到下方的 Query History 中刚刚所执行的创建语句变为了绿钩，这表明我们的语句是成功执行了的，我们再点击左侧的刷新按钮，可以看到数据库中出现了我们所创建的 example_db（这个警告下面的过程中也可能会出现，像这里一样忽视就好）
  • 接着我们指定使用刚刚创建的数据库example_db
USE example_db;
  • 然后创建表students，而后点击左边我们之前创建的 example_db 数据库
CREATE TABLE students (id INT,name STRING,age INT) PARTITIONED BY (class STRING) STORED AS PARQUET;
  • 进入后我们可以看到我们刚刚所创建的表students
  • 接着我们依次添加分区
ALTER TABLE students ADD PARTITION (class='A');
ALTER TABLE students ADD PARTITION (class='B');


  • 然后我们依次向表中添加数据，这里的第一条指令执行时就出现了之前说的可能会遇到的警告，同样也是可以忽略，下方的绿钩出现就说明该条语句已经执行成功了，第二条指令的执行为 success
INSERT INTO TABLE students PARTITION (class='A') VALUES(1, 'Alice', 20),(2, 'Bob', 21);
INSERT INTO TABLE students PARTITION (class='B') VALUES (3, 'Charlie', 22),(4, 'David', 23);


  • 然后我们可以查看当前数据库中所存在的表，这里我们只创建了students
SHOW TABLES;
  • 这里如果我们遇到了之前那样的警告而无法显示查询的结果时，重新执行几遍就可以了（查询语句重复执行没关系，而插入、建表这些语句重复执行就会导致数据的重复创建，所以之前只需要等待语句执行完毕的绿钩就可以了）
  • 我们也可以查看表 students 的结构
DESCRIBE students;
  • 接着我们查看表students的数据，可以看到我们之前所插入的数据
SELECT * FROM students;
  • 我们还可以查询特定分区数据
SELECT * FROM students WHERE class='A';
  • 进行聚合查询
SELECT class, COUNT(*) AS num_students FROM students GROUP BY class;
  • 对于查询结果，hue还可以对其进行可视化，并可以下载结果


二、Flume（官网）

1、简介

Flume是一个分布式的、可靠的、可用的服务，用于高效地收集、聚合和移动大量的日志数据。

2、Flume的核心概念

（1）一个flume agent（事务）可以分成三部分

  • 源（Source）：用于从外部系统（如日志文件、网络流、数据库等）获取数据，并通过将获取到的数据封装成 Event 发送到 Flume 的后续组件。
  • 通道（Channel）：用于临时存储从 Source 接收到的事件，直到它们被 Flume 的 Sink 组件处理。
  • 接收器（Sink）：用于将事件从 Channel 发送到外部系统（例如 HDFS、Kafka、数据库等）的组件。Sink 会从 Channel 中读取事件，并根据配置将其写入到目标系统中。

（2）flume 的工作流程

  • Source 从外部系统获取数据并将其转化为 Event。
  • Channel 存储这些 Event，确保数据可靠传输。
  • Sink 从 Channel 中读取 Event，并将其发送到外部存储或分析系统。

3、使用minikube部署flume

（1）向minikube中导入所需的镜像

minikube image load flume:latest
  • 进入minikube查看是否导入
minikube ssh
docker images

（2）由于这里展示的flume是集成到Hadoop中的，因此需要先使用hadoop-cluster.yaml文件部署Hadoop，部署后查看pod状态验证其是否正常运行

kubectl apply -f hadoop-cluster.yaml
kubectl get pods
  • 由于flume的日志是写在hdfs上的，因此，我们到hdfs上预先为其建立一个文件夹以存放flume日志
  • 我们先布置hdfs的端口转发，以在浏览器中访问到hdfs
kubectl port-forward svc/namenode 9870:9870
  • 在浏览器中输入如下地址
http://localhost:9870/
  • 点击这里可以进入文件管理界面
  • 接着我们回到命令行，进入namenode所在的pod，同样地，这里的namenode-764bfd689f-85pd7需替换为自己的pod名
kubectl exec -it namenode-764bfd689f-85pd7 -- bash
  • 而后在hdfs上创建文件夹并更改其权限
hdfs dfs -mkdir -p /flume/data
hdfs dfs -chmod 777 /flume

（3）使用flume-deployment.yaml文件部署flume，并查看pod状态验证其是否正常运行

kubectl apply -f flume-deployment.yaml
kubectl get pods
  • 创建完毕后我们可以进入flume所在的pod，以验证其是否能与hdfs连接，图中可以看到我们刚刚为flume建立的文件夹/flume/data
hdfs dfs -ls /

（4）测试flume的功能

  • 在flume的部署文件中，我们已经将/var/log/app/设置为了它的监听目录，因此我们在flume的pod中，写入一个文件到它所监听的这个目录下
for i in {1..5}; do
   echo "Text Test $i $(date '+%Y-%m-%d %H:%M:%S')" >> /var/log/app/text_test.log
done
  • 而后我们来到hdfs上存放flume数据的对应目录下，可以看到它已经自动生成了以日期命名的子文件夹
  • 我们进入这个文件夹，可以看到flume写入hdfs的.tmp临时文件
  • 这里我们需要静待一会，等待它写入完毕，此时.tmp后缀会消失
  • 而后我们回到命令行中的flume所在的pod中，从hdfs上下载flume写入的日志文件，这里的/2025-02-10/events-.1739222202785需要替换为实际的文件地址
hdfs dfs -get /flume/data/2025-02-10/events-.1739222202785 ./
ls
  • 接下来我们可以对比之前我们手动写入flume所监听的文件夹的文件与flume监听后写入hdfs的文件，是完全一致的，下面的events-.1739222202785也需要替换为自己实际的文件名
cat /var/log/app/text_test.log
cat events-.1739222202785
