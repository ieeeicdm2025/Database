    1、简介

    Hive是基于Hadoop的一个数据仓库工具，可以将结构化的数据文件映射为一张表，并提供类SQL查询功能。其本质是将HQL转化成MapReduce程序



    2、Hive的核心概念

    （1）Hive客户端

    • Hive CLI——本地
    • JDBC/ODBC——远程

    （2）Metastore

    • 是一个中心化的元数据存储，用于存储表、数据库、分区、列、数据类型等的元数据，通常使用关系型数据库（如内嵌的Derby、MySQL、PostgreSQL）来存储元数据

    （3）HiveServer2

    • 负责解析并处理来自客户端的SQL查询，并将查询转化为MapReduce作业或Spark作业，最后通过执行引擎将查询结果返回给客户端。它提供了JDBC和ODBC支持，使得各种外部工具（如BI工具、Java应用程序等）能够通过标准的数据库连接协议与Hive交互
    • beeline是Hive的命令行客户端工具，用于连接到 HiveServer2 并执行 HiveQL 查询。它支持JDBC连接，可以通过指定JDBC URL来连接到Hive服务。

    （4）Driver

    • 负责接收HiveQL查询请求，并将这些请求转化为内部表示，以便进一步处理

    3、使用minikube部署Hive

    （1）制作并向minikube中导入所需的镜像

    • 我们可以使用dockerfile来制作所需镜像
    docker build -f dockerfile_hive -t hive:latest .
    • 并拉取所需的另一个镜像
    docker pull mysql:oraclelinux9
    • 如果出现拉取错误的问题，也可以选择直接导入实验所提供的打包好的镜像
    docker load -i hive.tar
docker load -i mysql.tar
    • 然后向minikube中导入所需的镜像
    minikube image load apache/hive:latest
minikube image load mysql:oraclelinux9
    • 进入minikube查看是否导入，并在查看后输入exit退出minikube
    minikube ssh
docker images
exit


    （2）部署hive

    • 先使用mysql-deployment.yaml文件部署mysql
    kubectl apply -f mysql-deployment.yaml
    • 由于mysql需要初始化，因此我们需要等候一会，再使用hive-deployment.yaml文件部署Hive
    kubectl apply -f hive-deployment.yaml
    • 查看pod状态，确认已正常运行，这里需要注意，hive是集成到hadoop集群中的，因此需要先部署Hadoop集群的环境
    kubectl get pods
    • 使用kubectl的端口转发功能，以访问Hive的webui，这个转发命令行界面需要一直保持这个转发状态，停止转发后就无法访问webui了
    kubectl port-forward deployment/hiveserver2 10002:10002
    • 通过下面的url在浏览器访问Hive的webui

    http://localhost:10002/

    • 由于hive是集成到hadoop环境中的，因此我们也可以布置Hadoop集群中的端口转发以访问HDFS和YARN
    kubectl port-forward svc/namenode 9870:9870
kubectl port-forward svc/resourcemanager 8088:8088

    http://localhost:9870/

    http://localhost:8088/

    CREATE DATABASE IF NOT EXISTS example_db;

    CREATE TABLE students (id INT,name STRING,age INT) PARTITIONED BY (class STRING) STORED AS PARQUET;

    （6）进行专利文献数据查询cite75_99.txt

    • 创建表格patent_citation
    CREATE TABLE patent_citation (
        citing INT,
        cited INT
    )
    ROW FORMAT DELIMITED
    FIELDS TERMINATED BY ','
    LINES TERMINATED BY '\n'
    STORED AS TEXTFILE
    TBLPROPERTIES ('skip.header.line.count'='1');
    • 导入data1.txt文件中的数据
    LOAD DATA INPATH '/input/data1.txt' INTO TABLE patent_citation;
    • 查询一 被引用列表
    SELECT 
    cited, 
    concat_ws(',', collect_set(cast(citing AS string))) AS citing_list
FROM 
    patent_citation
GROUP BY 
    cited limit 10;


    • 查询二 被引用次数
    SELECT 
    cited, 
    COUNT(*) AS citing_count
FROM 
    patent_citation
GROUP BY 
    cited limit 10;


    • 查询三 被引用次数直方图
    SELECT 
    ct, 
    COUNT(*) AS group_count
FROM (
    SELECT 
        cited, 
        COUNT(*) AS ct
    FROM 
        patent_citation
    GROUP BY 
        cited
) cite_count
GROUP BY 
    ct;




    （7）进行专利文献数据查询apat63_99.txt

    • 创建表格patent_metadata
    CREATE TABLE patent_metadata (
    PATENT STRING,
    GYEAR INT,
    GDATE INT,
    APPYEAR INT,
    COUNTRY STRING,
    POSTATE STRING,
    ASSIGNEE STRING,
    ASSCODE INT,
    CLAIMS INT,
    NCLASS INT,
    CAT INT,
    SUBCAT INT,
    CMADE INT,
    CRECEIVE INT,
    RATIOCIT DOUBLE,
    GENERAL DOUBLE,
    ORIGINAL DOUBLE,
    FWDAPLAG DOUBLE,
    BCKGTLAG DOUBLE,
    SELFCTUB DOUBLE,
    SELFCTLB DOUBLE,
    SECDUPBD DOUBLE,
    SECDLWBD DOUBLE
)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
LINES TERMINATED BY '\n'
STORED AS TEXTFILE
TBLPROPERTIES ('skip.header.line.count'='1');
    • 导入data2.txt文件中的数据
    LOAD DATA INPATH '/input/data2.txt' INTO TABLE patent_metadata;
    • 查询四 年份或国家专利数统计
    SELECT 
    GYEAR, 
    COUNTRY, 
    COUNT(*) AS patent_count
FROM 
    patent_metadata
GROUP BY 
    GYEAR, 
    COUNTRY limit 10;
